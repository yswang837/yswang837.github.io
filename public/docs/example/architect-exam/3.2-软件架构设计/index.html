    1.软件架构的概述 #

    1.1 软件架构的概念 #

      软件架构即软件体系结构，软件架构设计就是需求的再分配，即将满足需求的职责分配到组件上。它搭建起了需求分析与软件设计之间的鸿沟。如下图：

    • 架构的本质
        1. 软件架构为软件系统提供了一个结构、行为和属性的高级抽象。
        2. 软件架构风格是特定应用领域的管用模式，架构定义了一个词汇表和一组约束。\
    • 架构的作用
        1. 软件架构是项目干系人进行交流的手段。
        2. 软件架构是可传递和可复用的模型。
        3. 软件架构使推理和控制的更改更加简单，有助于循序渐进的原型设计。
