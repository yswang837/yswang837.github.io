    1 安全基础基础 #

    1.1 对称与非对称加密 #

    一般用对称加密原文，已到达高速、节省空间的目的，一般用非对称加密保证数据的安全性。

    • 对称加密：公钥和秘钥一样，或者说公钥可以推导出秘钥。

    • 非对称加密：公钥和秘钥不一样，或者说公钥不能推导出秘钥。

      非对称加密：加密解密规则，用接收方的公钥加密。

      pa加密sa可解密，sa加密pa可解密。

    1.2 数字签名 #

      数字签名是采用非对称加密技术，具体用发送方A的私钥Sa加密信息后，接收方如果用发送方的公钥Pa能解密，那么这个信息就一定是A发送的。

    1.3 信息摘要 #

      相当于做md5，原文发生一小点变化，那摘要信息将完全不一样(雪崩效应)，下载安装包的时候，也会有个校验md5，就是指的这个。

    2 网络安全 #

    2.1 网络各层次的安全保障 #

      需要知道各个层用什么协议来保障安全。

      PGP (Pretty Good Privacy):优良保密协议。

      SSL (Secure Sockets Layer):安全套接字协议。

      TLS (Transport Layer Security):传输层安全协议。

      SET(Secure Electronic Transaction):安全电子交易协议。

      IPSEC(Internet Protocol Security):互联网安全协议。
